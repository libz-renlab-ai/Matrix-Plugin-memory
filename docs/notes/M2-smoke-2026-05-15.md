## Benchmark — 1000 rules

```
bench 1000 rules: p50=4ms p95=12ms mean=6.3ms
```

Well under the spec target P95 < 200ms (~16× headroom). Cost driven primarily
by 384-dim cosine scan over 1000 BLOBs in Node; embedding cost amortized via
LRU cache (size 64) when queries repeat.

## Known issues / follow-ups

- **Vitest worker segfault on shutdown**: After all tests pass, onnxruntime
  worker pool cleanup occasionally segfaults. Tests themselves pass; exit
  code is reported 0. Cosmetic; investigate in a follow-up.
- **`@xenova/transformers` install requires China-friendly registry**: First
  install via default npmjs.org failed with ECONNRESET. `--registry=https://registry.npmmirror.com`
  succeeded. README updated; consider documenting `.npmrc` recommendation.
- **Model weights ~125MB downloaded on first embedText call**: caches under
  `~/.cache/huggingface/`. SessionStart preheat is deliberately not added
  to keep hook startup fast; lazy load on first match works.

## Architecture / spec deviations

None major. Plan was followed; the only deliberate deviation is in `match.cjs`:

- Kept a synchronous `runMatchSync` back-compat export for legacy callers
  (only the M1 vitest test). Production hooks all use async `runMatch`.
